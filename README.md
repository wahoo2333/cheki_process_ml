# Cheki Process ML（重构版）

> 目标：从传统 CV 管线升级为 **实例分割驱动** 的拍立得检测与裁剪系统，采用方案 A：
> **Mask → 四角拟合（RANSAC / 可微拟合）→ 透视矫正输出**。

---

## 1. 项目目标与范围

### 1.1 核心目标

构建一个在真实拍摄条件下鲁棒的拍立得（或类似矩形相纸）自动裁剪系统，覆盖：

1. **检测并分割目标实例**（拍立得区域）
2. **从 mask 稳定恢复四个角点**
3. **执行透视变换得到标准化输出图**
4. 输出可用于后续 OCR / 检索 / 展示的高质量结果

### 1.2 先做什么（本阶段）

本仓库当前仅做项目初始化，不进入训练与推理开发。当前阶段产出：

- 项目技术路线确定（方案 A）
- 工程目录设计
- 数据集与标注规范
- 里程碑与验收指标

---

## 2. 方案选择：为什么是 A（实例分割）

### 2.1 与传统 CV 的差异

传统边缘/霍夫/轮廓方法在如下场景容易不稳定：

- 背景纹理复杂、边缘干扰多
- 部分遮挡、强反光、高噪声
- 拍摄角度极端导致边缘不完整
- 相纸边框与背景对比弱

方案 A 的核心思路是：

- 先让模型学习“哪一块是相纸”（mask）
- 再从 mask 拟合“几何结构”（四角）

这种“语义 + 几何”解耦方式通常对复杂场景更鲁棒。

### 2.2 模型选择（UNet 路线）

你提到希望使用“最新 UNet”。实践上建议采用 **UNet 系列可替换架构**（如 UNet / UNet++ / attention UNet / transformer encoder + UNet decoder），通过统一训练框架做可插拔实验。

> 重点不是先押注某个单体网络，而是先保证：
> - 数据闭环
> - 标注一致性
> - 角点拟合稳定性
> - 评估指标完整

## 4. 数据集需求（你需要准备什么）

### 4.1 数据规模建议（第一阶段）

最小可行版本（MVP）：

- **2,000～5,000 张图像**（先跑通）
- 场景覆盖优先于数量堆叠

推荐目标（更稳）：

- **8,000～20,000 张图像**

### 4.2 数据来源构成

建议混合来源：

1. **真实拍摄数据（主力）**
   - 手机多机型
   - 室内/室外
   - 近景/远景
2. **用户历史数据回流（脱敏后）**
3. **困难样本专项采集**（定向补齐）

### 4.3 必须覆盖的场景维度（非常关键）

每个维度都要“有样本”：

- 光照：强光、背光、夜景、阴影
- 角度：正视、轻透视、极端透视
- 遮挡：手指、贴纸、反光遮挡
- 背景：木纹、布料、杂物、图案干扰
- 目标状态：完整、边缘轻微缺失、边框磨损
- 多目标：同图 2～5 张拍立得
- 尺寸分布：大目标、中目标、小目标
- 清晰度：清晰、轻微模糊、运动模糊

### 4.4 数据切分建议

按“拍摄会话/设备”维度切分，避免同源泄漏：

- Train: 70%
- Val: 15%
- Test: 15%

> 注意：同一次连拍或同一视频抽帧，必须放在同一 split。

## 5. 标注规范（实例分割 + 几何信息）

### 5.1 标注任务定义

每个拍立得实例需标注：

1. **实例分割 mask（必须）**
2. **四角点（强烈建议）**：按顺时针统一顺序
   - `top_left`
   - `top_right`
   - `bottom_right`
   - `bottom_left`

> 若标注平台不方便直接标角点，可先多边形 mask，后续脚本从 polygon 近似提角。

### 5.2 类别定义

- 类别数：1（`polaroid`）
- 忽略区域（可选）：`ignore`（例如严重反光看不清边界）

### 5.3 标注边界规则（统一口径）

- 边界以“相纸外轮廓”为准（包含白边）
- 若边缘有小缺口，按“理想连续边界”补齐
- 遮挡不超过 25% 时，按可见轮廓+几何一致性推断
- 遮挡超过 25% 且无法确定边界：标为 `ignore` 或剔除

### 5.4 角点顺序规则（必须严格）

所有实例统一顺序：

1. top-left
2. top-right
3. bottom-right
4. bottom-left

并保证：

- 不可交叉
- 不可逆时针/随机顺序
- 允许亚像素坐标（若工具支持）

## 6. 推荐标注流程（可直接执行）

### 步骤 1：准备标注工具

可选工具：CVAT / Label Studio / Roboflow（任选其一，优先团队熟悉工具）

### 步骤 2：建立项目与标签

- 标签：`polaroid`
- 可选标签：`ignore`
- 任务类型：Instance Segmentation（Polygon）

### 步骤 3：先做 200 张试标

- 2 位标注员并行
- 交叉复核 30%
- 发现分歧，先更新“标注手册”再继续大规模标注

### 步骤 4：质检（QA）

每批次抽检 10～20%，检查：

- 漏标率
- 边界贴合度
- 角点顺序一致性
- 多目标是否全部标出

### 步骤 5：导出格式

建议优先导出 **COCO Instance Segmentation**（通用）

需要额外字段（可放 `attributes` 或独立 json）：

- `corners`: `[[x1,y1],[x2,y2],[x3,y3],[x4,y4]]`
- `quality`: `normal/hard/ignore`

## 7. 训练与推理的技术路线（先设计，不开发）

### 7.1 训练阶段（规划）

1. 输入图像 → UNet 输出实例 mask
2. 损失函数：Dice + BCE（可扩展 Focal）
3. 数据增强：透视、亮度、模糊、遮挡仿真

### 7.2 后处理阶段（规划）

1. 取最大连通域（或多实例逐个处理）
2. 轮廓提取与平滑
3. 四角拟合：
   - 方案 A1：RANSAC + 直线交点
   - 方案 A2：可微四边形拟合（后续可研究）
4. 角点排序与几何合法性校验
5. 透视变换输出标准图

### 7.3 回退机制（规划）

当角点拟合失败时：

- 回退到 `minAreaRect` 近似
- 输出置信度与失败原因码，便于线上监控

## 8. 评估指标（上线前必须定义）

建议至少包含：

1. **Mask IoU / Dice**（分割质量）
2. **Corner Error（像素）**（四角偏差）
3. **Homography 后重投影误差**
4. **最终裁剪成功率**（业务指标）
5. **困难场景分组指标**（背光/遮挡/反光等）

上线门槛建议（示例，后续可调整）：

- Test Dice ≥ 0.95
- 平均 Corner Error ≤ 5 px（按目标尺度归一化更合理）
- 端到端裁剪成功率 ≥ 98%
